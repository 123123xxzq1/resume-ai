> 你投出去的每 100 份简历，大概只有 25 份会被真人看到。其余 75 份死在了一个你看不见的对手手里——**ATS 简历筛选系统**。

## 什么是 ATS？

ATS 全称 Applicant Tracking System（申请跟踪系统），是企业用来**自动收集、过滤、评分**求职简历的软件。北美 99% 的《财富 500 强》、国内 80% 以上的中大型企业（包括 BAT、字节、美团）都在用。

常见的 ATS 系统：
- 国内：Moka、北森、e-HR、牛客企业版
- 海外：Workday、Greenhouse、Lever、Taleo、iCIMS

## ATS 不是"关键词匹配器"那么简单

很多人误以为 ATS 就是"看你简历里有没有 JD 里的词"。**真实机制是 5 层漏斗：**

### 第 1 层：格式解析（Parsing）

ATS 首先要把你的简历"看懂"，即从各种格式的文件里提取结构化字段：
- 姓名 / 联系方式
- 工作经历（公司、职位、起止时间、描述）
- 教育背景
- 技能清单

**如果解析失败，你的简历连被评分的机会都没有。**

**✅ 解析友好的做法：**
- 用**单栏排版**，避免双栏、侧边栏、表格布局
- 文件格式用 **PDF（文本型，非图片型）**或 docx
- 字体用 Arial、Helvetica、思源黑体等标准字体
- **不要用图标代替文字**（"📞 13800138000" 常被识别失败）
- 工作经历按"公司名 — 职位 — 时间"的顺序写

**❌ 常见死法：**
- Canva / Figma 模板导出的花哨 PDF，有时整段文字被解析成乱码
- 用文本框布局的 Word —— 解析后内容顺序会全乱
- 头像、LOGO 等图片信息

### 第 2 层：关键词匹配

这一层才是你熟悉的"关键词"。ATS 会把 JD 的必要条件拆成几十个关键词，然后扫描你的简历中出现的次数和位置。

**权重高的位置：**
1. 职位标题（Title）> 2. 技能板块 > 3. 工作经历正文 > 4. 其他

**典型关键词类型：**
- **硬技能**：Python / React / Kubernetes / SQL / 财务建模
- **工具**：Figma / Tableau / Jira / AWS
- **证书**：PMP / CFA / CPA / AWS Certified
- **方法论**：敏捷开发 / OKR / 数据驱动
- **软技能**（权重较低）：沟通能力 / 团队协作

### 第 3 层：经验匹配

ATS 会对齐 JD 里的硬指标：
- 工作年限（"3 年以上 React 经验"）
- 学历要求（"本科及以上"）
- 行业背景（"2 年以上 SaaS 经验"）

这里有个大坑：**ATS 经常按简单字符串匹配，而不是语义匹配**。

比如 JD 要"3 年以上 React 经验"，你写"精通 React, Vue, Angular, 5 年前端开发"，**ATS 可能识别不到 "3 年 React"**，因为没有显式出现这个时长。

**✅ 正确写法：**
> "5 年前端开发经验，其中 **4 年** 专注 React + TypeScript 技术栈。"

### 第 4 层：排序打分

ATS 按算法给每份简历打 0-100 分，HR 看到的是**按分数倒序排列的列表**。常见打分维度（以 Workday 为例）：

| 维度         | 权重 |
| ------------ | ---- |
| 关键词命中率 | 40%  |
| 经验匹配度   | 30%  |
| 教育背景     | 15%  |
| 量化成果     | 10%  |
| 简历完整度   | 5%   |

**低于阈值（通常 60 分）的简历，HR 根本看不到。**

### 第 5 层：HR 人工终筛

只有前 4 层的幸存者才能进入人类视野。这时 HR 平均只花 **7.4 秒** 扫一眼简历决定是否进入面试——所以**视觉清晰、重点突出** 同样重要。

## 让简历通过 ATS 的 7 个实操技巧

1. **标题用 JD 原词**：应聘"高级前端工程师"就写"高级前端工程师"，不要写"资深 Web 开发"。
2. **技能板块放顶部**：姓名联系方式下面，立刻列一个"核心技能"板块，把 JD 里的硬技能全塞进去。
3. **每段经历用动词开头**：主导、重构、优化、交付、提升 —— 比形容词更抓分。
4. **量化，量化，还是量化**：把"提升了性能"改成"将首屏加载时间从 3.2s 降到 1.1s，提升 65%"。
5. **标准化日期格式**：`2023.03 - 2024.11`，别写"近两年"、"大三时"。
6. **同义词全部列出**：`K8s / Kubernetes`、`AWS / 亚马逊云`，让 ATS 怎么搜都命中。
7. **投之前先自测**：用 [ResumeAI](/analyze) 把你的简历和 JD 粘进去，30 秒看到匹配度分数和缺失关键词。

## 常见误区

### ❌ "我用了花哨的模板，HR 看了印象深刻"

**真相：** HR 看到你之前，你要先通过 ATS。90% 的花哨模板会被 ATS 解析失败，你连上桌的机会都没有。

### ❌ "关键词塞满一整页就行"

**真相：** 现代 ATS（2022 后）已经用 NLP 识别"关键词堆砌"（Keyword Stuffing），过度堆砌会被**降分甚至标记**。

### ❌ "我的简历投出去没回音是因为没人看"

**真相：** 大概率是 ATS 把你筛掉了。同一份简历换个写法，面试邀约率可以从 3% 提升到 15%+。

## 总结

ATS 不是敌人，是规则。**看懂规则 → 按规则优化 → 打过 60 分阈值 → 进入 HR 视野**，这就是现代求职的铁律。
